spring简介？

  • Spring 是一个开源框架，由 Rod Johnson 创建，为简化企业级应用开发而生。Spring的核心是控制反转（ IoC ）和面向切面编程（ AOP ）。简单来说， Spring 是一个分层的 JavaSE / EE 一站式( Full-Stack )轻量级容器框架。

特性：

  • 轻量

  • 非入侵性

  • 容器

    1. 包含并管理应用对象的配置和生命周期

  • 框架

  • 一站式

    1. 可以在IoC和AOP基础上整合各种企业应用的开源优秀第三方类库以及本身也是一个框架级别。

  • 控制反转（Inversion of Control IOC）

    1. 促进了低耦合

IOC容器

描述

在 Spring IoC 容器读取 Bean 配置创建 Bean 实例之前，必须对它进行实例化。只有在容器实例化后，才可以从 IoC 容器里获取 Bean 实例并使用。

实现：

  • BeanFactory：IOC基本实现
  • ApplicationContext：BeanFactory接口的子接口
    • ClassPathXmlApplicationContext :ApplicationContext接口的实现类
    • 功能：从路径下加载配置资源
    • FileSystemXmlApplicationContext:ApplicationContext接口实现类
    • 功能：从文件系统加载配置资源

Bean的自动装配

  • 方法：Spring IoC 容器可以自动装配 Bean ，需要做的仅仅是在 的 autowire 属性里指定自动装配的模式即可
  • byType:（根据类型自动装配）：若 IoC 容器中有多个与目标 Bean 类型一致的 Bean ，在这种情况下， Spring 将无法判定哪个 Bean 最合适该属性，所以不能执行自动装配。反之进行自动装配
  • byName:(根据名称自动装配）：必须将目标 Bean 的名称和属性名设置的完全相同。
  • constructor:(通过构造器自动装配）：当 Bean 中存在多个构造器时，此种自动装配方式将会很复杂，不推荐使用。

    优缺点

    在一般情况下，在实际的 Bean 配置文件里设置 autowire 属性进行自动装配将会装配 Bean 的所有属性，然而，若只希望装配个别属性时， autowire 属性就不够灵活了。

Bean的继承(parent=”父类bean”)

  • Spring 允许继承 Bean 的配置，被继承的 Bean 称为父 Bean ，继承这个父 Bean的 Bean 称为子 Bean
  • 子 Bean 从父 Bean 中继承配置，包括 Bean 的属性配置
  • 子 Bean 也可以覆盖从父 Bean 继承过来的配置

可以类比Java面向对象中继承的概念，在bean的继承中,子类继承父类，可以完全继承父类的所有配置，当然子类自己也可以重写配置，并且该重写记录会覆盖父类。

Bean的抽象

  • 父 Bean 可以作为配置模板，也可以作为 Bean 实例。若只想把父 Bean 作为模板，可以设置 的 abstract 属性为 true ，这样 Spring 将不会实例化这个 Bean。
  • 并不是 元素里的所有属性都会被继承，比如： autowire 、 abstract 等。
  • 也可以忽略父 Bean 的 class 属性，让子 Bean 指定自己的类，而共享相同的属性配置，但此时 abstract 必须设为 true。

同样可以类比Java面向对象中抽象的概念，抽象体只提供相关的方法，具体的实现由继承他的子类去完成，当然，Bean作为抽象时，可以当作一个bean配置模板，当其中abstracts=true时，容器将不会对它进行实例化。

属性值 说明
singleton 默认值，每个IoC容器只创建一个Bean实例，且在IoC容器初始化时即创建Bean的实例
prototype 每次请求创建一个Bean实例，且在第一次调用getBean方法时才创建实例
request 一次HTTP请求中，一个Bean定义对应一个实例
session 一个HTTP Session中，一个Bean定义对应一个实例
globalSession 在一个全局的HTTP Session中，一个Bean定义对应一个实例

在 Spring 中，可以在 元素的 scope 属性里设置 Bean 的作用域。默认情况下， Spring 只为每个在 IoC 容器里声明的 Bean 创建唯一一个实例，整个 IoC 容器范围内都能共享该实例，所有后续的 getBean() 调用和 Bean 引用都将返回这个唯一的 Bean 实例。该作用域被称为 singleton ，它是所有 Bean 的默认作用域。

## 工厂与SPEL

SPEL

Spring 表达式语言（ SpEL ）：是一个支持运行时查询和操作对象图的强大的表达式语言。语法类似于 EL ， SpEL 使用 #{…} 作为定界符，所有在大框号中的字符都将被认为是 SpEL 。

通过 SpEL 可以实现：

  • 通过 Bean 的 id 对 Bean 进行引用
  • 调用方法以及引用对象中的属性
  • 计算表达式的值
  • 正则表达式的匹配
  • 字面量的表示：
  • 整数：
  • 小数：
  • 科学计数法：
  • String 可以使用单引号或者双引号作为字符串的定界符号： 或
  • Boolean：
  • 引用其他对象
  • 引用其他对象的属性
  • 引用其他对象的方法，还可以进行链式操作
  • 算术运算符：+、-、*、/、%、^
  • 加号还可以用作字符串连接
  • 比较运算符：<、>、==、<=、>=、lt、gt、eq、le、ge
  • 逻辑运算符号：and、or、not
  • if-else运算符：?:
  • 正则表达式：matches
  • 调用静态方法或静态属性：通过T()调用一个类的静态方法，它将返回一个Class Object，然后再调用相应的方法或属性。

示例

工厂方法创建Bean

调用静态工厂方法创建 Bean 是将对象创建的过程封装到静态方法中，当客户端需要对象时，只需要简单地调用静态方法，而不需要关心创建对象的细节。

  1. 要声明通过静态工厂方法创建的 Bean ，需要在 Bean 的 class 属性里指定拥有该工厂的方法的类，同时在 factory-method 属性里指定工厂方法的名称，最后使用 元素为该方法传递方法参数。

  1. 调用实例工厂方法创建Bean是将对象的创建过程封装到另一个对象实例的方法里，当客户端需要请求对象时，只需要简单的调用该实例方法而不需要关心实现细节。

    要声明通过实例工厂方法创建的 Bean ，需要在 Bean 的 Factory-bean属性里指定拥有该工厂的方法的Bean，同时在 factory-method 属性里指定工厂方法的名称，最后使用 元素为该方法传递方法参数。

  1. spring中两种不同的bean一种是普通bean，一种工厂bean（FactoryBean）其返回的对象不是指类的一个实例，而是该工厂的getObject方法所返回的对象
